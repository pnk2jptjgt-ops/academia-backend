# دليل النشر على Railway — خطوة بخطوة

هذا الدليل يفترض أنك ما سبق تستخدم Railway أو GitHub، ويشرح كل خطوة بالتفصيل.

---

## الخطوة 1: رفع الكود على GitHub

1. أنشئ حساب على https://github.com (مجاني)
2. أنشئ "مستودع" (Repository) جديد، اسمه مثلاً `academia-backend`
3. ارفع كل ملفات هذا المجلد لذلك المستودع
   - لو ما تعرف كيف، اطلب من Claude يشرح لك خطوات `git` بالتفصيل وقت التنفيذ

## الخطوة 2: إنشاء حساب Railway

1. روح لـ https://railway.com
2. سجّل حساب جديد (يفضّل تسجل بحساب GitHub مباشرة، يسهّل الربط)

## الخطوة 3: إنشاء المشروع

1. من لوحة Railway، اضغط **New Project**
2. اختر **Deploy from GitHub repo**
3. اختر المستودع اللي رفعته بالخطوة 1

## الخطوة 4: إضافة قاعدة البيانات

1. بنفس المشروع، اضغط **New** → **Database** → **PostgreSQL**
2. Railway ينشئ قاعدة البيانات ويولّد `DATABASE_URL` تلقائياً
3. هذا المتغيّر يتربط تلقائياً بالخدمة (الباك إند) — ما تحتاج تنسخه يدوياً

## الخطوة 5: تشغيل الترحيل (Migration) لأول مرة

1. من إعدادات الخدمة بـRailway، روح لـ **Settings** → **Deploy**
2. أضف أمر ما بعد البناء (Build Command أو Deploy Command):
   ```
   npx prisma migrate deploy && npm run prisma:seed
   ```
   هذا يجهّز كل جداول قاعدة البيانات + يضيف المدرسين الأساسيين تلقائياً

## الخطوة 6: التأكد من نجاح النشر

1. روح لتبويب **Deployments** وتابع السجلات (Logs)
2. لو شفت رسالة `الخادم يعمل على المنفذ ...` يعني نجح النشر
3. Railway يعطيك رابط عام مثل: `https://academia-backend.up.railway.app`
4. جرب فتح `https://[رابطك]/health` بالمتصفح — لو رجع `{"status":"ok"}` يعني كل شي تمام

## بعد النشر

- كل مدرس يقدر يسجّل دخول بالبريد المذكور بـ `seed.js` (مثلاً amir@academia.local) وكلمة المرور المؤقتة `ChangeMe123!`
- **مهم جداً**: لازم تغيّر كلمات المرور فوراً — حالياً ما فيه واجهة لتغييرها، فهذا أول شي نضيفه بالخطوة الجاية
